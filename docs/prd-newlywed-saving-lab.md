# 신혼절약연구소 MVP PRD

## 🎯 핵심 정보

**목적**: 신혼부부가 실생활에서 바로 적용할 수 있는 절약 팁을 카테고리별로 제공하여 생활비 절감을 돕는다
**사용자**: 결혼 1~3년 이내 신혼부부, 생활비 절약에 관심 있는 부부

## 🚶 사용자 여정

```
1. 메인 페이지 (/)
   ↓ 추천 팁 확인

2. [사용자 선택]

   [옵션 A] 추천 팁 클릭 → 팁 상세 페이지 → 읽기 완료 → 메인/전체 팁으로 이동
   [옵션 B] "전체 팁 보기" 클릭 → 전체 팁 리스트 페이지

3. 전체 팁 리스트 페이지 (/tips)
   ↓ 카테고리 필터 선택 (식비/공과금/통신비/보험/구독)

4. 필터링된 팁 목록 확인
   ↓ 특정 팁 클릭

5. 팁 상세 페이지 (/tips/[id])
   ↓ 상세 내용 확인

6. [완료] → 다시 전체 팁으로 이동 또는 다른 팁 탐색
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 추천 팁 조회 | 노션 DB에서 "추천여부" 필드가 true인 팁만 조회하여 메인에 표시 | 사용자에게 검증된 고품질 팁을 우선 제공 | 메인 페이지 |
| **F002** | 전체 팁 리스트 조회 | 노션 DB의 모든 절약 팁을 최신순으로 조회 및 표시 | 모든 팁 콘텐츠 접근성 제공 | 전체 팁 리스트 페이지 |
| **F003** | 카테고리 필터링 | 식비/공과금/통신비/보험/구독 카테고리별로 팁 필터링 | 사용자가 관심 분야의 팁만 빠르게 찾을 수 있음 | 전체 팁 리스트 페이지 |
| **F004** | 팁 상세 조회 | 선택한 팁의 전체 내용, 예상절약금액, 난이도, 태그 등 상세 정보 표시 | 팁을 실행하기 위한 구체적 정보 제공 | 팁 상세 페이지 |
| **F005** | Notion API 연동 | @notionhq/client를 사용한 노션 DB 데이터 실시간 조회 | CMS로 노션을 활용하여 개발 없이 콘텐츠 관리 | 모든 페이지 (데이터 레이어) |

## 📐 구현 상세

### Notion API 연동 구조

**환경 변수 (`.env.local`)**
```
NOTION_API_KEY=secret_xxxxx
NOTION_DATABASE_ID=xxxxx
```

**API 라우트 구조**
- `lib/notion.ts` - Notion Client 초기화 및 유틸리티 함수
- `app/api/tips/route.ts` - 전체 팁 조회 API (선택적)
- 서버 컴포넌트에서 직접 Notion API 호출 권장 (Next.js 16 App Router)

**데이터 조회 패턴**
- 추천 팁: `database.query()` + filter `is_recommended = true`
- 전체 팁: `database.query()` + sorts `created_at desc`
- 카테고리 필터: `database.query()` + filter `category = 선택값`
- 팁 상세: `pages.retrieve()` + `blocks.children.list()`

### 반응형 브레이크포인트
- **Mobile**: < 640px (sm)
- **Tablet**: 640px ~ 1024px (md ~ lg)
- **Desktop**: > 1024px (xl)

### 카드 컴포넌트 구조
```tsx
<Card>
  <CardHeader>
    <Badge>{category}</Badge>
    <CardTitle>{title}</CardTitle>
  </CardHeader>
  <CardContent>
    <div>예상 절약: {expected_savings}원/월</div>
    <div>난이도: {difficulty}</div>
  </CardContent>
</Card>
```
